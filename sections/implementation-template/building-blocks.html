FDO records

Specify how an FDO record is realized by the given FDO implementation.

[I3-1] Map FDO record → implementation structure
Starting with the conceptional FDO record containing attributes a1,...,an, map this record to an implementation structure that contains implementation-specific attributes a'1,...,a'n.
[I3-2] Map implementation structure → FDO record
Starting with an implementation structure that contains implementation-specific attributes a'1,...,a'n, map this structure to a conceptional FDO record containing attributes a1,...,an.

FDO Profiles

[I4-1] Profile representation
Each profile is itself an FDO. You must specify how the FDO profile is represented in the given implementation (i.e., how the FDO looks like; this can be understood as a recipe for building FDO profiles in the given FDO implementation).
Note: In the validations chapter, you need to define an algorithm for a machine to read and interpret a profile in the given implementation in order to perform validation of records against the profile.
[I4-2] Profile definition profile (Optional)
Give the PID for the profile definition profile from your implementation.
[I4-3] Root profile (Optional)
Give the PID for the root profile from your implementation.
[I4-4] Profile extension (Optional)
Specify how the extension mechanism of profiles works within your implementation.

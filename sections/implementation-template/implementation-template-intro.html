While all FDOs follow one common concept to make data FAIR, they can still be implemented using various different technologies. In order to guarantee interoperability between existing FDO implementations, it is essential that each implementation clearly specifies how it maps the technology-agnostic concepts (given in this specification) into a technology-specific implementation. Only then, interoperability between different implementations can emerge, i.e., a sequence of steps how to map an FDO in one implementation to an FDO in another implementation can be defined.

Each FDO implementation MUST specify how it implements the technology-agnostic FDO requirements. To this end, FDO implementations MUST provide details on

  1. how their FDOs implement the building blocks in the chosen technology/implementation, and
  2. how their FDOs can be processed automatically by a machine, i.e., a sequence of steps that defines at least how the type, the data and the metadata of the FDO can be extracted.

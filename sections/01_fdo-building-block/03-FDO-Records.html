FDO Records

Each FDO Record MUST fulfill the following requirements:

An FDO Record consists of tuples (called “FDO Attribute”):
[R3-1] FDO Record Structure
A PID identifying an FDO resolves into a structured information record, called the FDO Record.
[R3-2] FDO Attribute Tuple
The FDO Record consists of a finite number of tuples (called FDO attributes, short: attributes) that can be interpreted by a machine. The limit for the number of tuples per record depends on the implementation.
An FDO Record contains the attributes 0.FDO/Type, 0.FDO/Profile and 0.FDO/Data:
[R3-3] Profile Reference Attribute
Each FDO Record MUST link to at least one FDO Profile, which it complies to, via the designated attribute 0.FDO/Profile in its FDO Record. To link multiple FDO profiles, the attribute 0.FDO/Profile MAY be repeated multiple times in the FDO Record.
  • The first element of such a tuple representing the FDO's profile is either the PID 0.FDO/Profile , or the string FDO_Profile.
  • The second element is a PID referencing an FDO profile.
[R3-4] Type Attribute
Each FDO MUST specify its FDO type via the designated attribute 0.FDO/Type in its FDO Record. To link multiple FDO types, the attribute 0.FDO/Type MAY be repeated multiple times in the FDO Record.
  • The first element of such a tuple representing the FDO's type is either the PID 0.FDO/Type , or the string FDO_Type.
  • The 0.FDO/Type attribute follows the attribute referencing mechanism as defined in .
[R3-5] Data Attribute
Each FDO MUST contain the designated attribute 0.FDO/Data in its FDO Record at least once. The 0.FDO/Data attribute points to data associated with the FDO.
  • The first element of such a tuple representing the FDO's data is either the PID 0.FDO/Data , or the string FDO_Data.

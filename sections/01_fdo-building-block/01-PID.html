PID

A reliable PID System is a core component of all FDOs. It MUST provide referential stability over very long periods. A PID is an identifier that MUST resolve PIDs to an information structure which contains properties of the FDO. We will call this information structure an FDO Record. The combination of PIDs and FDO Records acts comparable to a passport which not only consists of an identification string, but also of properties which characterise the object. The PID System MUST be able to cope with the special requirements of the digital era which are frequent copying and moving of the associated bit-sequences and changing content in many cases. The PID system is required to handle frequent updates of PID records and provide consistent results to PID resolution requests to guarantee stable referencing to the bit-sequence. For such a system to be acceptable in the realm of FDOs the following criteria MUST be fulfilled:

[R1-1] PID Structure Specification
The structure of the PID MUST be based on specifications that are being maintained by an independent international organization. The structure MUST cater for a distributed solution.
[R1-2] PID System Governance
The PID system MUST be governed by an independent, non-profit organisation with global governance and proven sustainability for long periods.
[R1-3] Global PID Resolution
The PID system MUST support global resolution and therefore global uniqueness of the PIDs at a high level of stability.
[R1-4] FDO Record Resolution
The PIDs must resolve into a well-defined information structure called FDO Record which contains type-value pairs and supports the predictability and machine-actionability of its content.
[R1-4]
The FDO Records the PIDs are resolving to MUST be managed by independent trustworthy organizations.
[R1-6] PID Access Protocol
Resolution of the PIDs and therefore access to the FDO Records MUST be possible by a well-defined access protocol.
[R1-7] PID Security
The PID system MUST offer a high degree of security that only enables the owner of a FDO Record to manipulate it. It MUST be possible to restrict the access to the whole record or to individual key-value pairs.
By adhering to these requirements the PID system is supporting data sovereignty, transparency/traceability, accountability and persistence.

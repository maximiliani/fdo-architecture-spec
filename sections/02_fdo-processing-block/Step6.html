S-6: FDO Profile(s) Inspection

This step determines whether the FDO record’s profile(s) is the same or compatible with the FDO profile the client expects to process. This step assumes that [Step 4] was completed and that its status indicates that a 0.FDO/Profile field is present.

This step is in some way analogous to the FDO Type inspection step with the difference that a profile specifies the set of fields that a FDO Record should have to describe a particular type of objects.

Inputs
This step expects that Step 4 had a FDO Root Profile validation status of COMPLETE, REQUIRED_FIELDS, PROFILE_ONLY and two inputs:
  1. The list of Profiles that the client is looking for. This is called the expected FDO Profile list.
  2. The Profile_Matching flag. The three possible values are:
    • ALL_MATCH: Indicates that any FDO Profile is acceptable. This ignores the Expected FDO Profile list if provided.
    • EXACT_MATCH. Indicates that the FDO Profile matching requires an exact match with the FDO Profile(s) in the expected FDO Profile list.
    • DERIVED_MATCH. Indicates that the FDO Profile in the FDO Record can be derived Profile from the expected FDO Profile list. Derived profiles remain backward compatible with their parent profile.
Outputs
This step returns a flag describing the status of the match as well as a list with all of the FDO Profiles that match or are derived from the expected types. If the list is empty, it indicates that no matches, and if requested, no derived matches were found.
Expected behavior
In this step, the client will compare the list of FDO Profile in the FDO Record with the expected FDO Profile list for matches. The following lists the possible behaviors:
  1. The Expected FDO Profile List is empty and or the Profile_Matching is set to ALL_MATCH.

    The result is the FULL_MATCH flag with a list with all FDO Profiles from the FDO Record.

  2. The FDO Profile(s) in the FDO records fully match those in the expected FDO Profile List.

    The result is FULL_MATCH and the list of all expected FDO Profile List.

  3. The FDO Profile(s) in the FDO record match at least one of the FDO Profiles in the expected FDO profile list.

    The result is PARTIAL_MATCH with a list containing the FDO Profile(s) that match.

  4. The FDO Profile in the FDO records match none of the FDO Profile list. There are two different results depending on the the matching condition:
    • If the Profile_Matching_Flag is set to EXACT_MATCH, the client concludes that the FDO does not have the expected profile. The result is NO_MATCH and an empty list.
    • If the Profile_Matching_Flag is set to DERIVED_MATCH, the client will respectively analyse each of the FDO Profiles in the FDO Record and determine if they are derived from profiles that are within the expected FDO Profile list. Note that derived FDO profiles are compatible with their parent FDO profile. To determine is and FDO Profile is derived from another, the client resolves each of the FDO Profiles into their respective FDOs and then analyses these FDOs to see if the Profiles definition specifies that they are derived from FDO Profiles that match a profile within the expected FDO profile List. If the FDO Profiles within the FDO Record are derived from FDO Profiles in the expected FDO profile list, the FDO Record is compatible with one or more of its expected FDO Profile. Any errors in accessing FDO Profile’s FDO terminates the search for derived matches and results in that FDO Profile’s not matching. The results can be:
      • NO_MATCH and an empty list if no derived Profiles are found.
      • DERIVED_MATCH a list with the derived Profile matches.

The examples below assume the simple FDO Record described below. Adding to the example is the fact that the FDO Profile 21.32/234 is derived from FDO Profile 21.32/123 which is itself derived from FDO Profile 21.1/0.

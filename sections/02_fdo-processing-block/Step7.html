S-7: FDO Profile(s) Validation

From [Step 6], one or more FDO Profiles of interest were found in the FDO Record.

The FDO Profile validation step verifies that the FDO Record is valid according to the FDO Profile’s definition. An FDO is said to be an instance of a FDO Profile when it exhibits all the characteristics it specifies.

The validation step acquires the FDO Profile Definition for each of the FDO Profiles in the FDO Record by resolving each of the FDO Profiles’ identifiers and accessing their respective FDO. Each FDO Profile Definition specifies the properties and more specifically the set of attributes that must be present in any FDO that instantiates it. This definition includes such things as the names of the attributes, value type, description etc.

The profile validation returns a description of the state of validation which can range between complete and partial success and total failure indicating to the processing entity the validity of FDO Record and the values it can expect to find in it.

Inputs
The inputs consist of the FDO Record, and the list of the FDO Profile to validate the FDO Record against.
Outputs
The output consists of a list of Validated_Profile. A Validated_Profile consists of the following:
  • Profile PID: PID of the FDO Profile validated
  • Status:
    • VALID: All profile required attributes are found. Note that missing optional fields is acceptable.
    • PARTIALLY_VALID: Some profile required attributes are missing.
    • INVALID, No profile required attributes are found.
    • NOT_DEFINED. No FDO Profile definition could not be found.
  • Attributes: The list of attributes found in the FDO Profile. Empty list when invalid.
  • MissingAttributes: The list of missing required attributes. Empty list when VALID.
Expected behavior
The FDO Profile Validation shall process each FDO Profile selected from the FDO Record during step 6 and create a list of Profile Validation Status specifying how the FDO Record conforms to each of the FDO Profile’s requirements.

For each FDO Profile selected, the process will acquire its respective FDO and extract from it the FDO Profile Definition. From this definition it will get the list of attributes and look for them in the FDO Record. Every attribute it finds in the FDO Record will be added to the Attributes list. Every required attribute not found in the FDO Record is added to the MissingAttributes list.

The following describe the expected behavior for a given FDO Profile.

  1. If the FDO Profile cannot be mapped to a valid FDO Profile Definition the process throws one of the following Exception:
    • ERROR_RESOLVING_FDO: Profile PID cannot be resolved.
    • ERROR_PROFILE_NOT_ACCESSIBLE: The PID resolve but the FDO is not accessible / not valid.
    • ERROR_NO_PROFILE_DEF: No definition found in the Profile FDO.
    • ERROR_INVALID_PROFILE: The Profile FDO is not a Profile definition.
    • ERROR_PARSING_DEFINITION: Illegal or incompatible profile definition.
  2. The FDO Profile has an FDO Profile Definition and the FDO Record has all of the required attributes. The Profile_Validation_Status contains:
    • Status: VALID
    • Attributes: contains the list of all found attributes
    • MissingAttributes: empty.
  3. The FDO Profile has an FDO Profile Definition and the FDO Record has some of the required attributes. The Profile_Validation_Status contains:
    • Status: PARTIALLY_VALID
    • Attributes: contains the list of found attributes.
    • MissingAttributes: contains the required attributes that are missing.
  4. The FDO Profile has an FDO Profile Definition and the FDO Record has none of the required attributes. Status: INVALID Attributes: contains the list of any optional attributes. MissingAttributes: contains the list of required Attributes

S-4: FDO record Root Profile Validation

This step validates the FDO record according to the FDO root profile, specifically, it verifies that all required FDO Root Profile type value pairs are present within the FDO record. This method returns an integer status reflecting the degree of validation.

Inputs:
This step expects the output of S-3: exactly one well-structured FDO record. The input MUST NOT be blank.
Outputs:
A flag status reflecting the success, partial success, and error finding FDO root profile attributes.
Expected behavior:

This step finds the FDO root profile required type-value pairs. It compiles a list of all attributes within the FDO record. It then finds all the required attributes as specified by the Root Profile. This step specifically looks for the attributes: “0.FDO/Type”, “0.FDO/Profile”, and “0.FDO/Data” that are defined within the FDO root profile.

The Status value MAY have the following values:

  • COMPLETE (integer value of 0) to indicate that all the FDO root profile fields were successfully found. With such a status, the FDO processing has all the information it needs to proceed to the next step.
  • REQUIRED-FIELDS (integer value of 1) to indicate that only the 0.FDO/Profile and 00.FDO/Type attributes were found.
  • PROFILE_ONLY (integer value of 2) indicates that only the 0.FDO/Profile attribute(s) were found.
  • TYPE_ONLY(integer value of 3) indicates that only the 00.FDO/Type attribute(s) were found.
  • NON_COMPLIANT(integer value of -1) indicates that the core FDO Root profile attributes were not found. At this point, the FDO record cannot be further processed as an FDO.

FDO Records

Each FDO Record MUST fulfill the following requirements:

[R3-1] FDO Record Structure

A PID identifying an FDO resolves into a structured information record, called the FDO Record.

[R3-2] FDO attribute tuples

The FDO Record consists of a finite number of tuples (called FDO attributes, short: attributes) that can be interpreted by a machine. Conceptually, every tupel consists of a single key and a single value. Keys are NOT unique in an FDO record. The limit for the number of tuples per record depends on the implementation.

[R3-3] Minimal FDO Record Requirements

All FDO Record MUST contain the attributes from the 0.FDO/Root profile: 0.FDO/Profile, 0.FDO/Type and 0.FDO/Data

[R3-4] 0.FDO/Profile attribute

Each FDO Record MUST link to at least one FDO Profile, which it complies to. This is done using the designated 0.FDO/Profile attribute in its FDO Record. To link multiple FDO Profiles, the attribute 0.FDO/Profile MAY be repeated multiple times in the FDO Record.
  • The first element of such a tuple representing the FDO's profile is either the PID 0.FDO/Profile or the string FDO_Profile.
  • The second element is a PID referencing an FDO profile.

[R3-5] 0.FDO/Type attribute

Each FDO MUST specify its FDO Type using the designated 0.FDO/Type attribute. To link multiple FDO types, the attribute 0.FDO/Type MAY be repeated multiple times in the FDO Record. 0.FDO/Type follows the attribute referencing mechanism.
  • The first element of such a tuple representing the FDO's type is either the PID 0.FDO/Type or the string FDO_Type.
  • According to the attribute referencing mechanism, the value is the PID of another attribute in the same FDO Record.

[R3-6] 0.FDO/Data attribute

Each FDO MUST contain the designated 0.FDO/Data attribute at least once. The attribute 0.FDO/Data is used to reference the FDO's data. It uses the attribute referencing mechanism to reference the FDO's data.
  • The first element of such a tuple representing the FDO's data is either the PID 0.FDO/Data or the string FDO_Data.
  • According to the attribute referencing mechanism, the value is the PID of another attribute in the same FDO Record.

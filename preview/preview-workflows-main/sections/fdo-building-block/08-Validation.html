Validation is a key characteristic of FDOs. It enables detecting errors in the FDO Record early on. Validation SHOULD at least be carried out upon creation and update of an FDO.

An FDO Record is said to be valid if and only if

[R8-1]
it contains the attribute 0.FDO/Profile at least once. The value of each of these attributes MUST be references to profiles.
[R8-2]
For each of these referenced profiles,
  • the FDO Record MUST contain attributes that instantiate the attribute definitions listed in the profile, according to their cardinalities.
  • the values of these attributes MUST comply with the specified validation rules, e.g., with syntax rules, profile referencing rules, attribute referencing rules, or external vocabulary referencing rules, as given by the attribute definition.
[R8-3]
All attributes with an attribute definition MUST be correctly validated according to their validation rules.
[R8-4]
The FDO Record MUST successfully validates against the 0.FDO/Root profile.
[R8-5]
The FDO MUST NOT violate any of the rules specified in this document.

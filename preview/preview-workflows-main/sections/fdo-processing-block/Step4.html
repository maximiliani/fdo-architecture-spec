This step validates the FDO record according to the FDO Root Profile, specifically, it verifies that all required FDO Root Profile type value pairs are present within the FDO record. This method returns an integer status reflecting the degree of validation.

Inputs:
This step expects the output of S-3: exactly one well-structured FDO record. The input may not be blank.
Outputs:
A flag status reflecting the success, partial success, and error finding FDO Root Profile Attributes.
Expected behavior:

This step finds the FDO Root Profile required type-value pairs. It compiles a list of all attributes within the FDO record. It then finds all of the required attributes as specified by the Root Profile. This step specifically looks for the attributes: “0.FDO/Type”, “0.FDO/Profile”, “0.FDO/Name”, “0.FDO/Description” and “0.FDO/Data_Attributes” that are defined within the Root FDO. While the “0.FDO/Name” and “0.FDO/Description” are useful for human consumption, they are not necessary for machine processing of the FDO.

The Status value MAY have the following values:

  • COMPLETE (integer value of 0) to indicate that all the FDO Root Profile fields were successfully found. With such a status, the FDO processing has all the information it needs to proceed to the next step.
  • REQUIRED-FIELDS (integer value of 1) to indicate that only the 0.FDO/Profile and 0.FDO/Type attributes were found.
  • PROFILE_ONLY (integer value of 2) indicates that only the 0.FDO/Profile attribute(s) were found.
  • TYPE_ONLY(integer value of 3) indicates that only the 0.FDO/Type attribute(s) were found.
  • NON_COMPLIANT(integer value of -1) indicates that the core FDO Root values were not found. At this point, the FDO Record cannot be further processed as an FDO.

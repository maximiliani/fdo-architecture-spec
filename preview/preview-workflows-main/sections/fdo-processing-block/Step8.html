The step requires that [Step 7] successfully validated one or more profiles resulting in one or more sets of attributes. For each set of attributes, the FDO Attribute Processing checks each attribute’s type - value pair to determine if the value is appropriate given its type. This validation process will greatly vary in complexity depending on the specific type.
Inputs
The inputs consist of the list of Validated_Profile from [Step 7] with a status of VALID, PARTIALLY_VALID. . Note that this step could be used with any attribute list that would need validation.
Outputs
The output will be a list of ValidatedProfile where each attribute will have a ValidityStatus status and a ErrorDescription description.
Expected behavior
The FDO Attribute Processing iterates over the list of attributes and using the following steps:

  1. If it has a validation rule for the type it applies it on the attribute’s value. It will set the validity status and any error description it may have generated.

    The ValidityStatus will include, but not limited to, the following list:

    • VALID: Indicates that the value is appropriate according to the type validation rules.
    • INVALID: Indicates that the value did not comply with the validation rules.
    • PARTIAL: Indicates that the value is partially valid. An example is the case of a 0.FDO/DateTime attribute, where the date is filled in but there is no time information. While incomplete, this value could still be of interest to the client.
  2. It is expected that there could be additional specific type defined statuses.

Granularity

The granularity of FDOs depends on the pragmatic utility decisions made within communities of practice. To represent resources as FDOs, it is necessary to determine useful entities and their relationships. The granularity of FDOs is determined by the decision which resources are represented as separate FDOs and which are not. This decision is influenced by the need to balance detail and manageability in representing and manipulating the resources. Whatever the initial decision is, it can be revised later. One option is breaking up the bit sequence(s) of a given FDO into parts and creating new FDOs to represent these parts. An example is MRI scans of the brain. One option is to create one FDO for the entire set of slices. Another option is to create one FDO for each slice, with attributes indicating which slice it is.

[R2-1] Single Resource Representation

Each FDO represents exactly one resource. The resource MAY be complex, that is, it MAY consist of multiple other resources.

[R2-2] Collection Definition

If an FDO represents a resource which consists of multiple parts that are interpreted as resources in their own right, this FDO is called a collection. The decision which parts are interpreted as resources in their own right (and hence MAY be represented as their own FDOs) is dependent on pragmatic utility decisions within communities of practices.

[R2-3] Resource Description

A resource MUST be described by FDO attributes. It MAY have associated data bit-sequence(s), if it is a digital resource.

[R2-4] Attribute Scope

Every FDO attribute in an FDO record only describes the identified resource and not any other resources. Even if the FDO is a collection, all attributes in the FDO record describe theresource as a whole and parts of it. To describe one of the parts of the resource with FDO attributes, extract this part into its own FDO record, define an Inline FDO, or use a Composite Attribute .

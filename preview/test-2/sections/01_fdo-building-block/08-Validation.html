Validation

Validation is a key characteristic of FDOs. It enables detecting errors in the FDO Record early on. Validation SHOULD at least be carried out upon creation and update of an FDO.

[R8-1] Presence of the 0.FDO/Profileattribute

An FDO Record is said to be valid if and only if it contains the attribute 0.FDO/Profile at least once. The value of each of these attributes MUST be references to profiles. This is expressed using profile reference to 0.FDO/ProfileDef.

[R8-2] Profile Validation

For each of these referenced profiles,
  • the FDO Record MUST contain attributes that instantiate the attribute definitions listed in the profile, according to the their individual cardinalities.
  • the values of these attributes MUST comply with the specified validation rules, e.g., with syntax definition, profile referencing rules, attribute referencing rules, or external vocabulary referencing rules, as given by the attribute definition.

[R8-3] Attribute Validation

All attributes with an attribute definition MUST be correctly validated according to their validation rules.

[R8-4] Root Profile Validation

The FDO Record MUST successfully validate against the 0.FDO/Root profile.

[R8-5] Specification Compliance

The FDO MUST NOT violate any of the rules specified in this document.
